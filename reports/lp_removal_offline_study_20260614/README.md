## 判定结果（先给结论）

**不立项。死因 a)：信号无预测力（P1 不通过）。**

`liquidity_removal` 事件之后，ETH 价格**并不比随机时点更跌**——事件后做空的下行捕捉（MFE
~27bps）反而**低于无条件随机基线（~34bps）**；30 分钟收盘均值上价格甚至**微涨**（做空均值
亏 ~1bps；ETH/USDT 子集做空均值亏 ~5.7bps）。扣成本后秒级自动化够到的净 edge ≈ **−11bps**。
当年放弃这个执行链路是对的，但更根本的是：**这个信号本身（按此定义）没有可货币化的方向
预测力**，不是"被抢跑"也不是"薄于成本"，而是从源头就没有 edge。

## 第 2 部分：出块后剩余 edge 衰减曲线（生死线）

样本 = 4,222 个 ETH liquidity_removal 事件（≥$1,500，价格窗口完整）。方向 = 做空。
k=30min 窗口，mean/median 分列（bps）：

| τ（出块后） | 做空 MFE 均值 | MFE 中位 | 做空收盘均值 | 收盘中位 |
|---|---|---|---|---|
| t=0   | 27.54 | 17.45 | **−0.95** | −0.71 |
| +30s  | 27.48 | 17.49 | **−1.03** | −0.29 |
| +60s  | 27.45 | 17.44 | **−1.11** | −0.53 |
| +120s | 27.74 | 17.87 | −0.89 | −0.68 |
| +300s | 28.07 | 17.96 | −0.38 | −0.06 |

- **t=0 是否有 edge**：没有。做空收盘均值为**负**（价格微涨，做空亏），MFE 仅是波动产生的
  "任何入场点都有的"下行游走。
- **衰减多快 / τ=30~60s 还剩多少**：曲线**几乎水平**（27.5bps 从 0→300s 不变）——因为根本
  没有 edge 可衰减。τ=0/30/60s 数值几乎相同（**口径说明**：1m K 线无法分辨同一分钟内的
  0/30/60s，多落在同一根 bar；但既然各 horizon 包括 t=0 与 +300s 都无 edge，秒级分辨已无意义）。
- **做多方向对照**：做多收盘均值 = +0.95（t=0）/ +1.11（+60s）bps——即"撤离后价格略偏涨"，
  与"流动性撤离=看跌"的先验**相反**（但幅度极小，不构成做多信号）。

## 第 3 部分：基线对照 + 抢跑诊断 + 规模分层

### (3a) 无条件基线（随机入场，做空，k=30min）
| 组别 | 做空 MFE 均值 | 做空收盘均值 | n |
|---|---|---|---|
| **事件组（ETH）** | **27.5** | **−0.95** | 4,222 |
| 随机基线（ETH） | **33.98** | +0.11 | 6,000 |

- 事件组下行捕捉（27.5bps）**低于**随机基线（34bps）→ 信号相对随机是**负 edge**。
- 收盘差 Welch t = **−1.52**（要求 >2 才算显著超额下行）→ **P1 不通过**。事件后并不比随机更跌。

### (3b) 抢跑诊断（事件前价格）
- 事件前 5min→t0 收益均值 **+0.09bps**，前 1min→t0 **−0.05bps**（≈0）。
- **价格在事件前基本不动** → 这**不是"被抢跑后捡残羹"**（死因 b），而是**根本没有方向信号**
  （死因 a）。如果有真 edge 被抢跑，事件前应已明显下行；并未观察到。

### (3c) 规模分层（机制验证）
按 size 三分位的做空 MFE（τ=60s, k=30min）：t1=**32.1** → t2=**25.4** → t3=**24.9** bps。
- **越大的撤离，后续下行越弱**（单调**反向**）。这与"撤离越大→跌得越多"的机制**相反**，
  **不构成机制支持**（脚本里 P4 的 monotone 布尔为 True，但方向是反的，按预注册解读为机制
  不成立 / 反机制）。最大档多为自动化 ~$33M 机械再平衡，定义上无信息含量，符合该反向结果。

## 第 6 部分：诚实结论（Q1–Q6）

**Q1 样本量与窗口**：12 个月（2025-05-28→2026-05-28），ETH 有效事件 **4,222**（够统计，
甚至偏多）；但 ~96% 是 ETH/USDC 的自动化 ~$9–33M 机械再平衡（高度自相关，信息含量低）。
ETH/USDT 仅 167。**BTC = 0（V2 BTC 池已死，无法测 BTC）**。

**Q2 衰减曲线（生死线）**：**t=0 即无 edge**；曲线从 0→300s 水平（27.5bps MFE 全程不变，
收盘全程微负）。τ=30~60s "剩余"= 与 t=0 同样的**非 edge**，扣成本为负。没有"厚 t=0 被
60s 吃光"的结构——是压根没有。（1m 分辨率无法切 0/30/60s，但因各 horizon 皆无 edge，结论不受影响。）

**Q3 基线超额 + 抢跑**：事件组下行**不超**随机基线（反而更低），Welch t=−1.52，不显著。
事件前价格≈不动 → **不是残羹（死因 b），是真没信号（死因 a）**。

**Q4 成本账**：τ=30/60s 做空毛 ≈ −1bps，扣 10bps 成本后**净 ≈ −11bps**。秒级自动化即便
零延迟也无法把负的变正。

**Q5 立项判定 + 死因**：**不立项，死因 a（无预测力）**。给操作者的直接回答：
- 当年放弃**执行链路**是对的（人工延迟确实不可行）；
- 但更重要：**信号本身（按"任意 UniV2 Burn"这个定义）没有方向预测力**，不是"能做只是没法
  快"，而是"快了也没用"。主观印象"有效"大概率是确认偏误 / 选择性记忆。
- **唯一未被本研究证否的角落**：本定义把信号稀释进了海量 dust 与自动化再平衡。一个**更窄、
  更可能有信息含量**的定义（如：非自动化地址发起的、相对池 TVL 占比大的、伴随同向 swap 抽干
  的真实撤资）本研究**未单独隔离**——但这需要新的信号定义与新数据，属另一个假设，不能作为
  挽救本信号的理由。

**Q6 观察节**
- 数据本身揭示：UniV2 上"liquidity_removal"已被自动化做市/金库的机械 remove-add 主导，
  把它当方向信号是把噪声当信号。
- 规模反单调（越大越弱）进一步说明大额撤离更可能是机械再平衡而非恐慌出逃。
- 方法论限制（已如实标注）：① 1m 价格无法分辨秒级 τ；② 价格用交易所 perp、事件在链上，
  存在轻微时间错配（≤1 分钟）；③ 仅 4 个 V2 池、BTC 无样本，结论限于"ETH UniV2 Burn"这一
  具体定义；④ 公共节点 408 限流导致 WBTC 池未 100% 扫完（但已扫部分均为 0，结论稳健）。
